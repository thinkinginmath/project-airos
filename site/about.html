Our story

Why Project Airos exists

We believe AI should be sovereign: user-controlled, transparent, and available on devices people already own. Project Airos blends frontier research like Meta’s Project Aria with open-source values and community co-creation.

The challenge

AI ecosystems today are cloud-only, gated by proprietary APIs, and designed to capture data. This model leaves individuals, educators, and smaller labs without privacy, agency, or affordability.

Vendor lock-in: Closed platforms shape the rules and pricing.
Data risk: Personal context and sensor feeds must leave the device to become useful.
Equity gap: Emerging markets and public institutions can’t rely on always-on connectivity.

Our response

Airos is a modular Agent OS that runs locally or at the edge. It empowers builders to orchestrate voice, vision, and context-aware intelligence with open tooling.

Local-first: Keep computation and memory on trusted hardware whenever possible.
Portable memory: Combine short-term and long-term context under user ownership.
Community-driven: Build in the open, inspired by pioneers like Project Aria while extending what’s possible.

What makes Airos different

Meta’s Project Aria showed how rich sensor suites can power advanced AR experiences, yet the stack remained closed. Airos takes the inspiration and opens the blueprint: from the voice loop to the memory fabric, every layer is built to be forked, audited, and remixed.

Dimension Closed status quo Project Airos

Roadmap milestones

The core team and community contributors focus on three parallel tracks of development.

Voice-first interfaces

Voice UX toolkits, wake word inference, and multimodal prompt loops for daily wearables.

Memory + personalization

Portable vector stores and secure sync protocols for individuals, families, and classrooms.

Edge deployment program

Blueprints for running Airos in labs, schools, and community spaces with commodity hardware.
